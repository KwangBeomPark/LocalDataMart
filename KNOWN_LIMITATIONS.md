# 기술적 제약 사항 및 미지원 범위 (KNOWN_LIMITATIONS.md)

본 문서는 Finance DataMart Tool (MVP) 버전의 아키텍처적 한계점, 미지원 범위 및 제약 조건을 명시합니다.

---

## 🚫 1. 미지원 범위 (Out of Scope)
1. **단일 실행 파일 (exe) 미제공**: PyInstaller를 활용한 바이너리 패키징이나 별도 설치 프로그램 배포를 제공하지 않습니다. 로컬 파이썬 런타임 환경에서 기동해야 합니다.
2. **DuckDB 및 Parquet 파일 포맷 미지원**: 현재 버전은 성능 지연을 보완하기 위해 대규모 데이터베이스 엔진(DuckDB)이나 고성능 압축 포맷(Parquet)을 사용하지 않습니다. 모든 입출력은 CSV 및 Excel로만 구동됩니다.
3. **Power BI / ODBC / SharePoint 연동 미지원**: 외부 리포팅 플랫폼 연결이나 클라우드 데이터 공유(SharePoint)를 직접 제어하는 브릿지를 내장하고 있지 않습니다.
4. **설정 편집 GUI 미지원**: 데스크톱 UI 홈에서 설정(`config.xlsx`)의 내용물 컬럼 매핑 규칙을 직접 수정하는 UI 화면을 지원하지 않습니다. 설정 수정은 엑셀 프로그램을 직접 사용하여 갱신해야 합니다.
5. **GitHub Actions 및 CI/CD 워크플로우 미지원**: 깃허브 공개 리포지토리 상에서 코드가 변경될 때 자동으로 빌드하고 정합성을 체크하는 CI/CD 워크플로우 파이프라인은 설계되어 있지 않습니다. 무결성 검증은 로컬에서 `pre_release_check.py`를 수동 호출하여 확인해야 합니다.

---

## ⚠️ 2. 아키텍처적 한계 및 제약 (Technical Limitations)
1. **Pandas 인메모리 연산 한계**:
   - 데이터 가공을 위해 모든 엑셀 데이터를 로컬 PC의 RAM(메모리) 상에 DataFrame 형태로 일시 로드합니다.
   - 따라서 가용 메모리를 초과하는 대규모 파일(예: 단일 파일 10GB 이상) 처리 시 메모리 부족 오류(Out of Memory)가 발생할 수 있습니다.
2. **Reconciliation 정량 오차 임계치 (`0.01`)**:
   - 원본 데이터 합산액과 Clean Table 간의 부동 소수점 오차에 대한 허용 한계를 소수점 둘째 자리 미만인 `0.01`로 하드코딩해두었습니다.
   - 단 0.01의 수치 왜곡이라도 감지되면 정합성 실패(FAIL)로 가동이 중단됩니다.
3. **단일 스레드 처리**:
   - 병렬 연산 및 멀티프로세싱 프레임워크가 적용되어 있지 않아, CPU 코어를 단일하게 점유해 순차 처리하므로 수천 개 이상의 엑셀 파일 처리 시 속도가 선형적으로 증가합니다.
